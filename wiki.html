Ziphra Messenger Documentation

The Ziphra project is divided into three main components: Common (shared), Client (Android), and Server (multi-module backend). Each part of the project has a dedicated GitHub Wiki with documentation and technical details:

  • ziphra-common: Shared code for client and server.
  • ziphra-client: Android application (client-side).
  • ziphra-server: Server-side modules.

This repository contains multiple coordinated backend modules:

  • ziphra-core: Core logic and base classes shared across server components. Javadoc
  • ziphra-security: Common security logic used by ziphra-appserver and ziphra-messaging, including encryption, signature handling, and token management. Javadoc
  • ziphra-commonback: Utility classes and helpers used throughout the backend. Javadoc
  • ziphra-appserver: Main entry point for client connections. Manages session establishment, API endpoints, and authorization. Javadoc
  • ziphra-idsprovider: Responsible for generating secure, random user identifiers and nicknames for anonymous use. Javadoc
  • ziphra-sessionmanager: Handles in-memory sessions, key management, and temporary identity tracking. Javadoc
  • ziphra-messaging: Delivers push messages securely between users, preserving anonymity and end-to-end encryption guarantees. Javadoc
